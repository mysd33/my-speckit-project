# IPC API Contract: Photo Album Organizer

**Scope**: Electron IPC channels exposed via `contextBridge` from main process to renderer
**Transport**: `ipcRenderer.invoke` (renderer) ↔ `ipcMain.handle` (main)
**Preload bridge**: `window.api` (defined in `src/preload/index.js`)
**Date**: 2026-05-31
**Plan**: [../plan.md](../plan.md)

---

## General Conventions

- All channels use `invoke` (request/response); no `send`/`on` fire-and-forget.
- Every handler returns a **plain serialisable object** (no class instances, no `Buffer` objects except where noted).
- On error, handlers return `{ error: string }` — never throw to the renderer.
- All `id` fields are UUID v4 strings generated by `crypto.randomUUID()` in the main process.
- All `date` fields are ISO date strings `YYYY-MM-DD`.
- All timestamp fields are ISO 8601 UTC strings.
- `file_path` and `thumb_path` are relative paths from `userData`; the renderer constructs `app://` URLs as `app://${file_path}`.

## Albums

### `albums:list`

Returns all albums with their cover image path and photo count, grouped and sorted for the main page.

**Request**: _(no parameters)_

**Response**:
```ts
{
  data: Array<{
    id: string;
    name: string;
    date: string;           // YYYY-MM-DD
    sort_order: number;
    cover_thumb_path: string | null;   // thumb_path of first photo; null if album empty
    photo_count: number;
    created_at: string;
    updated_at: string;
  }>
}
// On error:
{ error: string }
```

**Behaviour**: Albums sorted by `date DESC`, then `sort_order ASC` within each date. The renderer groups by date on the client side.

---

### `albums:create`

Creates a new album.

**Request**:
```ts
{
  name: string;   // required, non-empty after trim
  date: string;   // YYYY-MM-DD
}
```

**Response**:
```ts
{
  data: {
    id: string;
    name: string;
    date: string;
    sort_order: number;
    cover_thumb_path: null;
    photo_count: 0;
    created_at: string;
    updated_at: string;
  }
}
// On error:
{ error: string }
```

**Validation errors**: `"Name is required"` | `"Invalid date format"`

---

### `albums:update`

Updates an album's name and/or date.

**Request**:
```ts
{
  id: string;
  name?: string;   // if provided, must be non-empty after trim
  date?: string;   // if provided, must be YYYY-MM-DD
}
```

**Response**:
```ts
{ data: { id: string; name: string; date: string; updated_at: string } }
// On error:
{ error: string }
```

---

### `albums:delete`

Deletes an album and all its photos. Also removes all files under `userData/photos/{album-id}/`.

**Request**:
```ts
{ id: string }
```

**Response**:
```ts
{ data: { deleted: true } }
// On error:
{ error: string }
```

---

### `albums:reorder`

Persists a new `sort_order` for a set of albums within a single date section. Called after a drag-and-drop completes.

**Request**:
```ts
{
  updates: Array<{ id: string; sort_order: number }>
}
```

**Response**:
```ts
{ data: { updated: number } }  // count of rows updated
// On error:
{ error: string }
```

**Behaviour**: All updates are applied in a single SQLite transaction. Only `sort_order` is modified; other fields are unchanged.

## Photos

### `photos:list`

Returns all photos for a given album, ordered by `sort_order ASC`.

**Request**:
```ts
{ album_id: string }
```

**Response**:
```ts
{
  data: Array<{
    id: string;
    album_id: string;
    filename: string;
    file_path: string;
    thumb_path: string;
    mime_type: string;
    file_size: number;
    sort_order: number;
    created_at: string;
  }>
}
// On error:
{ error: string }
```

---

### `photos:open-dialog`

Opens the OS native file picker and returns the selected file paths. No import is performed.

**Request**: _(no parameters)_

**Response**:
```ts
{ data: { file_paths: string[] } }   // empty array if user cancelled
// On error:
{ error: string }
```

**Behaviour**: Filters to `image/jpeg`, `image/png`, `image/gif`, `image/webp` (and common extensions). Multi-select enabled.

---

### `photos:import`

Imports one or more photos into an album. For each file the main process:
1. Validates mime type and file size (≤ 20 MB).
2. Copies the file to `userData/photos/{album_id}/{uuid}.ext`.
3. Generates a 300 px-wide JPEG thumbnail via `nativeImage` and saves to `userData/photos/{album_id}/{uuid}.thumb.jpg`.
4. Inserts a row into the `photos` table.

**Request**:
```ts
{
  album_id: string;
  file_paths: string[];   // absolute OS paths returned by photos:open-dialog
}
```

**Response**:
```ts
{
  data: {
    imported: Array<{
      id: string;
      filename: string;
      file_path: string;
      thumb_path: string;
      mime_type: string;
      file_size: number;
      sort_order: number;
      created_at: string;
    }>;
    rejected: Array<{
      original_path: string;
      reason: string;   // "Unsupported format" | "File too large (max 20 MB)" | "File not found"
    }>;
  }
}
// On error (entire operation failed):
{ error: string }
```

**Behaviour**: Partial success is allowed — successfully imported files return in `imported`, failed files return in `rejected`. The operation never partially imports a single file.

---

### `photos:delete`

Removes a single photo from an album. Deletes the row from SQLite and removes both the original and thumbnail files from `userData`.

**Request**:
```ts
{ id: string }
```

**Response**:
```ts
{ data: { deleted: true } }
// On error:
{ error: string }
```

## Preload Bridge (`window.api`)

The `contextBridge` exposes the following shape in `src/preload/index.js`:

```js
contextBridge.exposeInMainWorld('api', {
  albums: {
    list:    ()       => ipcRenderer.invoke('albums:list'),
    create:  (args)   => ipcRenderer.invoke('albums:create', args),
    update:  (args)   => ipcRenderer.invoke('albums:update', args),
    delete:  (args)   => ipcRenderer.invoke('albums:delete', args),
    reorder: (args)   => ipcRenderer.invoke('albums:reorder', args),
  },
  photos: {
    list:       (args) => ipcRenderer.invoke('photos:list', args),
    openDialog: ()     => ipcRenderer.invoke('photos:open-dialog'),
    import:     (args) => ipcRenderer.invoke('photos:import', args),
    delete:     (args) => ipcRenderer.invoke('photos:delete', args),
  },
});
```

The renderer accesses these via `window.api.albums.list()` etc. (wrapped by `src/renderer/src/api.js` for error-handling convenience).

## Security Notes

- `contextBridge` is used; `nodeIntegration` and `contextIsolation` are never disabled.
- The `app://` protocol handler validates that resolved paths do not escape `userData` via `path.relative` check before serving any file.
- IPC handlers validate all input fields before touching the filesystem or database.
